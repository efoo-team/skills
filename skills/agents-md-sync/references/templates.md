## 1. 共通の執筆規律

- **電報体（telegraphic style）で書く**。前置き・接続詞・丁寧語を削り、事実だけを短く書く。
- **全記述に根拠を持つ**。コマンドは実際に設定ファイル・CI 定義に存在するもの、パス・シンボルは実在するものだけを書く。根拠を確認できないが重要と思われる記述は「〔要確認: 理由〕」マーカー付きで書く。推測で断定しない。
- **節に書く内容が無ければ節ごと省略する**。テンプレートを埋めるために薄い内容を書かない。逆に行数を稼ぐための水増しもしない。
- **既存ファイルの人間由来の記述**（生成メタデータ行が無い、または前回生成後に人間が追記した決定・注意書き）は、コードベースと矛盾しない限り残す。矛盾する場合は削除せず〔要確認〕を付けてレポートに載せる。
- 時限性の記述（「YYYY年MM月以前は～」）を書かない。現行の状態だけを書く。

## 2. ルート AGENTS.md テンプレート

```markdown
# {プロジェクト名} — PROJECT KNOWLEDGE BASE

## OVERVIEW
{何のプロジェクトか + 中核スタック。2文以内}

## STRUCTURE
{高レベル地図。パッケージ・主要ディレクトリごとに1行、非自明な用途だけを注記。
 ファイル単位の列挙はしない}

## WHERE TO LOOK
| やりたいこと | 場所 | 補足 |
|---|---|---|
{タスク起点のナビゲーション表。「認証を触る」「バッチを追加する」等の実タスク単位}

## CODE MAP
| Symbol | 種別 | 場所 | 役割 |
|---|---|---|---|
{中心シンボル（エントリポイント・被参照の多い export）。LSP 等で測定できた場合のみ。
 測定できなければ節ごと省略し「centrality 未測定」とレポートに書く}

## CONVENTIONS
{デフォルト・言語標準から逸脱するものだけ。逸脱の根拠となる設定ファイルを併記}

## ANTI-PATTERNS（このプロジェクト固有）
{禁止 + 代替のペアで書く。出典（コメント・lint 設定・レビュー指摘）があれば併記}

## COMMANDS
{推測不能なビルド・テスト・生成コマンド。正確なフラグ付き。package.json 等から
 自明に分かるものは代表的なものに絞る}

## NOTES
{既知の落とし穴・環境の癖・必須環境変数}
```

## 3. サブディレクトリ AGENTS.md テンプレート

```markdown
# {ディレクトリパス} — AGENTS.md

## OVERVIEW
{この階層の責務。1行}

## STRUCTURE
{サブディレクトリが5超のときのみ。1行/ディレクトリ}

## WHERE TO LOOK
| やりたいこと | 場所 | 補足 |
|---|---|---|

## CODE MAP
{この階層の中心シンボル。測定できた場合のみ}

## CONVENTIONS
{親（ルート）と異なるもの・この階層にだけ適用されるものだけ。親と同じ内容は書かない}

## ANTI-PATTERNS（この階層固有）
{禁止 + 代替のペア}

## COMMANDS
{この階層固有の実行コマンドのみ（例: workspace 単位のテスト起動）}
```

**親との重複禁止**: 執筆エージェントは確定済みの親 AGENTS.md 全文を受け取り、親に既にある内容を書かない。親の内容を前提に差分だけを書く。

## 4. 生成メタデータ行

各ファイルの**末尾**に、次の HTML コメントを1行で記録する:

```
<!-- agents-md-sync: commit=abc1234 generated=2026-07-08 mode=update -->
```

- `commit` = 生成・更新時点の `git rev-parse --short HEAD`。次回実行時の drift 検出（scan-repo.sh）がこの値を基準に `git diff` を取る
- 既存ファイルを更新したときは必ずこの行も更新する
- HTML コメントにするのは、エージェントが本文として読む情報を汚さないため

## 5. 品質ゲート

| 項目 | 基準 |
|---|---|
| 行数 | ルート・サブとも**一律 1000 行以下**（efoo 設計原則の正本の上限に一致） |
| 必須節 | OVERVIEW と、生成メタデータ行。他の節は内容があるときのみ |
| 重複 | 子は親と同じ内容を持たない |
| 根拠 | 全コマンド・パス・シンボルが実在する。不確実な記述は〔要確認〕マーカー |
| 文体 | 電報体。一般論・訓辞なし |
